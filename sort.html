🔄 정렬 알고리즘

버블·삽입·퀵, 대표 정렬 3가지 방법

📍 정렬이란?

정렬(Sort)은 흩어져 있는 데이터를 일정한 순서대로 줄 세우는 작업이에요. 오름차순(작은 것부터)이나 내림차순(큰 것부터)으로 정리하죠.

🏫 일상 속 정렬
• 키 순서로 줄 서기
• 학번 순서로 출석부 정리
• 가나다순으로 단어 정리
• 가격 낮은 순으로 상품 정렬하기
💡 회전(Pass)이란?
정렬 알고리즘에서 자주 쓰는 표현이에요. "한 회전(1 회전)"이란 알고리즘이 배열을 한 번 훑는 단위를 말해요. 회전마다 결과가 어떻게 변하는지를 추적하는 게 정렬 학습의 핵심이에요!

정렬 알고리즘에는 여러 종류가 있는데, 여기서는 버블·삽입·퀵 정렬을 배워볼 거예요.

🃏 삽입 정렬 (Insertion Sort) 바로가기(영상)

1️⃣ 개념

삽입 정렬은 왼쪽부터 차례대로 보면서, 새 원소를 이미 정렬된 왼쪽 구간의 알맞은 위치에 끼워 넣는 방법이에요.

🃏 비유: 손에 든 카드 끼워 넣기
카드를 한 장씩 왼손의 정렬된 카드들 사이에 끼워 넣는 모습을 떠올려 보세요.
새 카드가 들어올 때마다 왼쪽으로 밀어 넣어 올바른 자리를 만들어요.
이게 바로 삽입 정렬이에요.

2️⃣ 직접 보기 (시뮬레이터)

아래 배열을 삽입 정렬로 정렬해 봐요. [▶ 다음 단계]를 눌러서 천천히 따라가 봐요.

시작 전
[▶ 다음 단계] 버튼을 눌러 시작해 보세요!
🎨 색깔 의미
🟧 주황(삽입할 값) | 🟦 파랑(비교/이동 중) | 🟩 초록(정렬 완료 구간)

3️⃣ 동작 원리 정리

  1. 1회전: [1] 인덱스 값을 왼쪽 정렬 구간([0])에 맞춰 끼워 넣기
  2. 2회전: [2] 인덱스 값을 왼쪽 정렬 구간([0]~[1])에 맞춰 끼워 넣기
  3. 3회전: [3] 인덱스 값을 왼쪽 정렬 구간([0]~[2])에 맞춰 끼워 넣기
  4. ...이런 식으로 정렬 구간이 점점 오른쪽으로 확장돼요.
📐 회전당 작업
매 회전마다 오른쪽의 새 원소 1개를 뽑아, 왼쪽의 정렬 구간에서 자기보다 큰 값들을 한 칸씩 밀고 알맞은 자리에 넣어요.

📌 예시 배열: 회전마다 끝까지 (시뮬레이터와 동일)

k회전 후는 [k] 인덱스 값을 왼쪽 정렬 구간에 끼워 넣은 직후의 배열 전체예요. 아래는 원소 8개 예시 [8, 3, 6, 1, 7, 2, 9, 4]예요. 큰 수·작은 수가 섞여 있어 회전이 지날수록 누가 어디로 밀리고 끼어드는지 따라가기 좋게 고른 배열이에요. (삽입 회전은 1~7까지)

구분배열 ([0]~[7])설명
초기[8, 3, 6, 1, 7, 2, 9, 4][0]만 길이 1짜리 정렬 구간
1회전 후[3, 8, 6, 1, 7, 2, 9, 4][1]의 3 삽입 → 8이 한 칸 밀림
2회전 후[3, 6, 8, 1, 7, 2, 9, 4][2]의 6이 3·8 사이로 → 8이 또 밀림
3회전 후[1, 3, 6, 8, 7, 2, 9, 4][3]의 1이 맨 앞으로 → 3,6,8이 연달아 오른쪽으로 밀림
4회전 후[1, 3, 6, 7, 8, 2, 9, 4][4]의 7이 6·8 사이로 삽입
5회전 후[1, 2, 3, 6, 7, 8, 9, 4][5]의 2가 앞쪽 깊숙이 → 큰 수들이 한꺼번에 밀림
6회전 후[1, 2, 3, 6, 7, 8, 9, 4][6]의 9는 자리 유지(5회전과 배열 동일)
7회전 후[1, 2, 3, 4, 6, 7, 8, 9][7]의 4가 3·6 사이 → 전체 오름차순
정렬 완료[1, 2, 3, 4, 6, 7, 8, 9]7회전 직후와 같음

4️⃣ 장점과 단점

✅ 장점
  • • 구현이 단순하고 직관적이에요
  • • 거의 정렬된 데이터에서 빨라요
❌ 단점
  • • 데이터가 많으면 느려요
  • • 최악의 경우 이동이 많이 발생해요

🫧 버블 정렬 (Bubble Sort) 바로가기(영상)

1️⃣ 개념

버블 정렬은 인접한 두 원소를 비교해서, 왼쪽이 더 크면 자리를 바꾸는 방법이에요. 한 번 끝까지 진행하면 가장 큰 값이 거품처럼 맨 뒤로 떠올라요!

🥤 비유: 탄산음료의 거품
탄산음료를 따르면 큰 거품이 위로 떠오르죠?
버블 정렬도 마찬가지예요. 한 회전을 돌면 가장 큰 값이 맨 뒤로 떠오르고, 두 번째 회전에서는 두 번째로 큰 값이 그 앞으로 떠올라요.
그래서 이름이 버블(거품) 정렬이에요!

2️⃣ 직접 보기 (시뮬레이터)

인접한 두 칸을 비교하면서 큰 값이 뒤로 가는 모습을 따라가 봐요.

시작 전
[▶ 다음 단계] 버튼을 눌러 시작해 보세요!
🎨 색깔 의미
🟦 파랑(비교 중인 두 칸) | 🟧 주황(자리 바꾸는 중) | 🟩 초록(정렬 완료)

3️⃣ 동작 원리 정리

  1. 1회전: [0]-[1] 비교 → [1]-[2] 비교 → ... → [n-2]-[n-1] 비교 (왼쪽이 크면 바꾸기). 끝나면 가장 큰 값이 맨 뒤로!
  2. 2회전: 다시 [0]부터 [n-3]까지 인접 비교 (맨 뒤는 이미 정렬됨). 끝나면 두 번째로 큰 값이 뒤에서 두 번째로!
  3. ...반복하면 점점 정렬된 영역이 뒤에서부터 늘어나요.
📐 회전당 작업
매 회전마다 인접한 두 원소를 비교하면서 끝까지 1번 통과해요. 한 번의 회전이 끝나면 그 회전에서 가장 큰 값이 맨 뒤(또는 정렬된 영역 바로 앞)로 이동해요.

📌 예시 배열: 회전마다 끝까지 (시뮬레이터와 동일)

k회전 후는 아직 정렬되지 않은 앞부분에서 인접 비교·교환을 k번째로 끝까지 한 번 통과한 직후의 배열이에요. 예시 배열 [8, 3, 6, 1, 7, 2, 9, 4]에서는 매 통과마다 큰 값이 뒤쪽으로 거품처럼 올라가는 모습이 잘 보여요. (원소 8개 → 이 구현에서는 7회전까지)

구분배열설명
초기[8, 3, 6, 1, 7, 2, 9, 4]정렬 확정 칸 없음
1회전 후[3, 6, 1, 7, 2, 8, 4, 9]9가 맨 뒤 [7]에 확정, 8·4도 뒤로 정리
2회전 후[3, 1, 6, 2, 7, 4, 8, 9]8이 [6]에 확정, 앞쪽은 계속 교환
3회전 후[1, 3, 2, 6, 4, 7, 8, 9]7이 [5]에 확정, 작은 값이 왼쪽으로 모임
4회전 후[1, 2, 3, 4, 6, 7, 8, 9]이미 전체 오름차순
5~7회전 후[1, 2, 3, 4, 6, 7, 8, 9]교환 없이 통과만 반복
정렬 완료[1, 2, 3, 4, 6, 7, 8, 9]시뮬레이터가 7회전 후 종료

4️⃣ 장점과 단점

✅ 장점
  • • 알고리즘이 매우 단순해요
  • • 거의 정렬된 데이터에 빨라요
❌ 단점
  • • 교환 횟수가 매우 많아요
  • • 데이터가 많으면 느려요

⚡ 퀵 정렬 (Quick Sort) 바로가기(영상)

1️⃣ 개념 — “기준 잡고, 반으로 나눠서 다시 한다”

퀵 정렬은 한 번에 전체를 정렬하려 하지 않아요. 대신 기준값(피벗)을 하나 세우고, “피벗보다 작은 것들은 왼쪽으로, 큰 것들은 오른쪽으로” 한 덩어리씩 정리해 나가요. 이렇게 구간을 나눈 뒤, 왼쪽 덩어리와 오른쪽 덩어리에 똑같은 방법을 반복해요. 이 큰 흐름을 분할 정복(나누고, 각각 해결하기)이라고 불러요.

이 페이지와 아래 시뮬레이터에서는 규칙을 하나로 고정했어요. 피벗은 항상 “지금 말하는 구간에서 맨 왼쪽 칸”의 값이에요. (프로그램에서는 그 구간의 첫 인덱스, 예를 들어 [0] 또는 구간이 줄어든 뒤의 [2] 같은 칸을 말해요.)

✂️ 비유: 반장 기준으로 줄 세우기
줄 맨 앞에 선 친구를 기준(피벗)이라고 하면, 키가 기준보다 작은 친구들은 왼쪽으로 모이고, 큰 친구들은 오른쪽으로 모이게 할 수 있어요.
그다음에는 왼쪽 무리만, 오른쪽 무리만 각각 또 맨 앞 사람을 기준으로 같은 방식을 반복하면, 결국 전체가 순서대로 정렬돼요.
🤔 많은 친구들이 여기서 막히는 이유
버블·삽입은 “한 줄을 차례대로” 보면 되는데, 퀵 정렬은 기준(피벗), 왼쪽·오른쪽에서 오는 두 포인터(i, j), 구간이 잘린 뒤 재귀적으로 같은 일이 한꺼번에 등장해요. 그래서 처음엔 난해하게 느껴질 수 있어요. 괜찮아요 — 먼저 “한 구간에서 피벗 한 번 제자리 잡기”만 따라가면 전체가 훨씬 보기 쉬워져요.

2️⃣ 큰 그림 — 딱 세 줄로 기억하기

  1. 피벗 정하기 — 지금 다루는 구간의 맨 왼쪽 값을 기준으로 삼는다.
  2. 한 번 나누기(partition) — 피벗보다 작은 값은 왼쪽으로, 큰 값은 오른쪽으로 값들을 모은다. 이때 피벗은 정렬했을 때 가야 할 자리로 옮겨 진다.
  3. 반복 — 피벗 왼쪽 구간과 오른쪽 구간에 대해, 또 1번부터 반복한다.
🔍 이 페이지의 “한 번 나누기” 안에서 하는 일 (i 와 j)
아래 시뮬레이터는 한 구간마다 두 개의 포인터 i, j를 써요. 헷갈리지 말고 역할만 기억하면 돼요.
  • i — 왼쪽에서 오른쪽으로 가면서, 피벗보다 큰 값을 찾아요. (피벗보다 작거나 같으면 계속 지나가요.)
  • j — 오른쪽에서 왼쪽으로 가면서, 피벗보다 작거나 같은 값을 찾아요. (피벗보다 크면 계속 지나가요.)
  • i와 j가 서로 아직 “교차하기 전”이면 — 찾은 두 값이 서로 자리가 안 맞는 거예요. 그래서 두 칸의 값을 서로 바꿔요.
  • i와 j가 만나거나 지나치면 — 이제 더 바꿀 필요가 없어요. 마지막으로 피벗(맨 왼쪽에 있던 값)과 j가 가리키는 칸을 바꿔서, 피벗을 최종 위치에 놓아요. 이 칸은 앞으로 다시 움직이지 않아요(시뮬레이터에서는 초록으로 표시).

💡 배열 칸 위의 [0], [1], …는 맨 왼쪽부터 0부터 시작하는 인덱스예요. 헷갈리면 숫자보다 색(주황 피벗, 파랑 비교)을 보면 돼요.

2-2️⃣ 딱 한 번의 partition, 눈으로 따라가기

말로만 들으면 헷갈리죠? 작은 예시 배열로 partition 한 번을 처음부터 끝까지 직접 따라가 볼게요.
예시 배열: [5, 3, 8, 1, 7] — 총 5칸, 인덱스 [0]~[4]

🎯 partition의 목표 딱 한 가지
이번 구간에서 피봇(5)의 왼쪽에는 5보다 작은 값만, 오른쪽에는 5보다 큰 값만 오도록 배열을 정리하고, 피봇(5) 자신은 최종 위치에 자리 잡는 것이에요.
단계 배열 상태 무슨 일이 일어나나요?
🔵 시작 [5, 3, 8, 1, 7]
피봇=5 (인덱스[0])   i=1, j=4 		피봇은 맨 왼쪽 값(5)으로 정해요.
i는 피봇 바로 오른쪽(인덱스 1)에서 출발, j는 맨 오른쪽(인덱스 4)에서 출발해요.
① i → 이동
(큰 값 찾기)		 [5, 3, 8, 1, 7]
i: 1 → 2에서 멈춤 		arr[1] = 3 → 3 ≤ 5 이므로 "이 값은 왼쪽에 있어도 돼요." → i 한 칸 이동
arr[2] = 8 > 5 → "8은 피봇보다 큰데 지금 왼쪽에 있어요!" → i 멈춤
② j ← 이동
(작은 값 찾기)		 [5, 3, 8, 1, 7]
j: 4 → 3에서 멈춤 		arr[4] = 7 → 7 > 5 이므로 "이 값은 오른쪽에 있어도 돼요." → j 한 칸 이동
arr[3] = 1 ≤ 5 → "1은 피봇보다 작은데 지금 오른쪽에 있어요!" → j 멈춤
③ 교환!
i=2, j=3
i < j → 아직 안 교차! 		교환 전: [5, 3, 8, 1, 7]
교환 후: [5, 3, 1, 8, 7]
i→3, j→2 		i가 가리키는 8(큰 값)과 j가 가리키는 1(작은 값)의 자리가 바뀌어 있어요.
→ 서로 자리를 바꿔요! 이제 1은 왼쪽으로, 8은 오른쪽으로 이동했어요.
교환 뒤 i++, j-- 로 각자 한 칸씩 더 이동.
④ 교차 확인
i=3, j=2
i > j → 종료!		 [5, 3, 1, 8, 7]
더 이상 바꿀 쌍 없음 		i(3) > j(2) → i와 j가 교차했어요! 이제 "자리가 잘못된 쌍"은 더 이상 없어요.
스캔·교환 반복을 끝내요.
⑤ 피봇 자리 확정! 🎉 교환 전: [5, 3, 1, 8, 7]
교환 후: [1, 3, 5, 8, 7] 		피봇(arr[0] = 5)과 arr[j=2] = 1을 교환해요.
이제 5는 인덱스[2]에 영구 정착! ✅
왼쪽 [1, 3] → 모두 5보다 작음
오른쪽 [8, 7] → 모두 5보다 큼
⚠️ i(=3)는 이 교환에 참여하지 않아요. 피봇은 j 위치하고만 교환해요.
💡 i와 j로 찾아서 바꾸는 이유가 뭔가요?
  • 🔵 i는 "피봇보다 큰데 왼쪽에 있는 값"을 찾아요. 이 값은 오른쪽으로 가야 해요.
  • 🔴 j는 "피봇보다 작은데 오른쪽에 있는 값"을 찾아요. 이 값은 왼쪽으로 가야 해요.
  • 🟡 둘이 멈추면? → "자리가 잘못된 두 값"을 동시에 찾은 거예요. 서로 바꾸면 한 번에 두 개가 올바른 쪽으로 이동해요!
  • ✅ i와 j가 교차하면 → 더 이상 자리 잘못된 쌍이 없어요. 마지막으로 피봇을 j 위치에 놓으면 피봇이 최종 자리를 잡아요.
❓ 교차 후, 피봇은 왜 i가 아니라 j 위치로 가나요?
교차했다는 건 i > j인 상태예요. 이 시점에서:
  • j가 멈춘 곳 → "피봇 이하인 값"의 마지막 위치예요. j까지(j 포함)는 모두 피봇보다 작거나 같아요.
  • i가 멈춘 곳 → "피봇 초과인 값"의 첫 위치예요. i부터는 모두 피봇보다 커요.
  • 즉 j가 경계선이에요. 피봇을 j 자리에 넣으면, 왼쪽은 전부 피봇 이하·오른쪽은 전부 피봇 초과가 딱 맞게 완성돼요.
  • i는 이 교환에 참여하지 않아요. i의 역할(큰 값 찾기)은 교차 순간 이미 끝났어요.
🔁 이제 이걸 왼쪽·오른쪽에 반복해요
위에서 5가 인덱스[2]에 자리 잡은 뒤, 왼쪽 구간 [1, 3]과 오른쪽 구간 [8, 7] 각각에 똑같이 partition을 해요. 구간이 점점 짧아지면서 결국 모든 숫자가 자기 자리를 찾게 돼요.

3️⃣ 직접 보기 (시뮬레이터)

아래에서는 피벗이 구간의 맨 왼쪽에 고정된 채로, 위에서 설명한 i / j 이동 → 필요하면 교환 → 피벗 자리 확정이 그대로 펼쳐져요. 처음에는 버튼만 누르며 주황(피벗)과 파랑(비교 중인 칸)만 따라가 보세요.

시작 전
[▶ 다음 단계] 버튼을 눌러 시작해 보세요!
🎨 색깔 의미
🟧 주황(피벗) — 이번에 제자리를 찾을 기준값
🟦 파랑(i/j) — 지금 비교하거나 바꿀 후보인 칸
🟩 초록 — 이번 단계에서 피벗의 최종 위치가 정해졌어요

4️⃣ 동작 원리 정리

  1. 정렬할 구간이 정해지면, 그 구간에서 맨 왼쪽 칸의 값을 피벗으로 삼는다.
  2. i는 왼쪽에서, j는 오른쪽에서 출발해 서로를 향해 온다. 규칙에 맞는 칸을 찾을 때까지 전진한다.
  3. i < j이면 두 칸의 값을 교환하고, 다시 같은 방식으로 스캔을 이어간다.
  4. i와 j가 만나거나 교차하면, 피벗을 j 위치와 바꿔 피벗의 자리를 확정한다.
  5. 피벗 왼쪽 구간과 오른쪽 구간을 각각 또 다시 퀵 정렬한다. 구간에 숫자가 하나뿐이면 할 일이 없다.
📌 버블·삽입과 다른 점 한 줄로
매 회전마다 “맨 뒤 한 칸 확정”이 아니라, 퀵 정렬은 피벗 하나가 자기 최종 자리에 앉는 것을 한 단위로 생각하면 편해요. 그다음은 잘린 왼쪽·오른쪽 구간에서 같은 일을 반복해요.

📌 예시 배열: 전체 → 전반부(왼쪽) → 후반부(오른쪽) (시뮬레이터와 동일)

퀵 정렬은 구간마다 partition 한 번이 끝날 때마다 피벗이 자리를 잡고, 배열이 왼쪽 덩어리와 오른쪽 덩어리로 갈라져요. 시뮬레이터는 왼쪽(전반부) 구간을 먼저 끝까지 처리하고, 그다음 오른쪽(후반부)으로 넘어가요.

예시 배열은 [5, 1, 6, 3, 9, 4, 2, 7](1~9 중 숫자 8개)예요. 첫 partition 뒤에도 오른쪽에 9, 6, 7처럼 크기가 뒤섞여 있어서, 후반부에서도 숫자가 확 바뀌는 과정을 시뮬레이터로 따라가기 좋게 골랐어요.

구간라운드배열 ([0]~[7])설명
① 전체 — 맨 처음 한 번 나누기
전체초기[5, 1, 6, 3, 9, 4, 2, 7]구간 [0]~[7], 피벗 = 5
전체1회전[4, 1, 2, 3, 5, 9, 6, 7]피벗 5 → 인덱스 [4] 확정. 왼쪽은 5보다 작은 값, 오른쪽은 큰 값으로 정리
② 전반부 (왼쪽 덩어리) — 구간 [0]~[3]부터 먼저 끝까지 (시뮬레이터 순서)
왼쪽1라운드[3, 1, 2, 4, 5, 9, 6, 7]구간 [0]~[3], 피벗 4 → [3]. 작은 값들이 왼쪽으로 모임
왼쪽2라운드[2, 1, 3, 4, 5, 9, 6, 7]구간 [0]~[2], 피벗 3 → [2]
이어서 구간 [0]~[1]에서 피벗 2로 partition하면 전반부 완료 → [1, 2, 3, 4, 5, 9, 6, 7] (아래 후반부는 이 직후부터)
③ 후반부 (오른쪽 덩어리) — 구간 [5]~[7] (9, 6, 7 정리)
오른쪽1라운드[1, 2, 3, 4, 5, 7, 6, 9]구간 [5]~[7], 피벗 9 → [7]. 9가 맨 뒤로, 6·7이 앞쪽으로 움직임
오른쪽2라운드[1, 2, 3, 4, 5, 6, 7, 9]구간 [5]~[6], 피벗 7 → [6]. 6과 7 순서 확정
—정렬 완료[1, 2, 3, 4, 5, 6, 7, 9]이 예시에는 숫자 8이 없음(1~9 중 8개만 사용)

5️⃣ 장점과 단점

✅ 장점
  • • 평균적으로 매우 빨라요
  • • 대규모 데이터에서도 효율적이에요
❌ 단점
  • • 구현이 버블/삽입보다 복잡해요
  • • 피벗 선택이 나쁘면 성능이 크게 떨어져요

구분 버블 정렬 삽입 정렬 퀵 정렬

1회전 후 가장 큰 값이 맨 끝[n-1] 인덱스에 확정 [1]까지 왼쪽 정렬 구간 확장(입력에 따라 배열이 달라짐) 전체 구간에서 partition 1번 → 피벗 1개 최종 위치 확정
2회전 후 뒤에서 두 번째까지 큰 값들이 자리 확정 [2]까지 정렬 구간 확장(값은 입력에 따라 1회전과 같을 수 있음) 왼쪽·오른쪽 덩어리로 나뉜 뒤, 스택 순서에 따라 다음 구간에서 partition 반복
