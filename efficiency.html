📍 효율성이란?

효율성(Efficiency)은 같은 일을 얼마나 빠르고 적은 자원으로 처리할 수 있는지를 말해요. 알고리즘에서는 보통 "몇 번을 비교하느냐"로 효율성을 측정해요.

🚗 비유: 학교 가는 길
집에서 학교까지 가는 길이 여러 개가 있다고 해볼게요. 어떤 길은 가깝지만 막히고, 어떤 길은 멀지만 뻥 뚫려있어요.
어떤 길을 골라야 가장 빠를까요? 상황에 따라 다르겠죠! 평일 등교 시간이라면 막히는 길을 피하고, 주말이면 짧은 길이 좋아요.
알고리즘도 똑같아요. 상황(데이터의 특성)에 맞게 골라야 가장 빨라요.

💡 핵심 포인트
• 데이터가 정렬되어 있지 않으면 → 선택의 여지 없이 선형 탐색 (이진 탐색은 사용 불가)
• 데이터가 정렬되어 있고 양이 많으면 → 이진 탐색이 압도적으로 빠름
• 데이터가 정렬되어 있어도 양이 매우 적으면 → 굳이 이진 탐색을 쓸 필요 없이 선형 탐색도 OK

🎬 실생활 속 상황별 알고리즘

여러 상황에서 어떤 알고리즘이 효율적일지 살펴봐요.

📚
도서관에서 책 찾기
10만 권의 책이 도서번호 순으로 정렬되어 있어요. 특정 책을 찾고 싶다면?
→ 이진 탐색 (정렬된 큰 데이터)
📋
신입 회원 명단 찾기
동아리에 새로 들어온 50명의 명단(가나다순 X)에서 특정 이름을 찾는다면?
→ 선형 탐색 (정렬되지 않음)
🛒
쇼핑몰 주문번호
매일 수백만 건의 새 주문(정렬되지 않음) 중에서 특정 주문번호를 찾는다면?
→ 선형 탐색 (정렬되지 않음)
🔍
검색 엔진 인덱스
10억 개의 정렬된 문서 ID에서 특정 ID를 찾는다면?
→ 이진 탐색 (정렬된 매우 큰 데이터)
📝
짧은 단어 목록
친구가 보내준 5개의 단어 중에서 특정 단어를 찾는다면?
→ 선형 탐색 (양이 적음)
🎓
전교생 학번 검색
전교생 1,200명의 학번이 학번 순으로 정렬된 명단에서 특정 학생을 찾는다면?
→ 이진 탐색 (정렬되었고 양도 많음)
