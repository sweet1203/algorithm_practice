🔍 탐색 알고리즘

데이터 안에서 원하는 값을 찾는 두 가지 방법

📍 탐색이란?

탐색(Search)은 여러 개의 데이터 중에서 원하는 값을 찾아내는 과정이에요. 우리가 매일 하는 일과 비슷해요!

🏫 일상 속 탐색
• 책장에서 특정 책 찾기
• 출석부에서 내 친구 이름 찾기
• 카카오톡에서 친구 검색하기
• 사전에서 단어 찾기
🔢 왜 인덱스는 0부터 시작할까요?
배열의 인덱스는 "시작 위치에서 얼마나 떨어져 있는가"를 나타내요. 첫 원소는 시작 위치에서 0칸 이동한 자리이기 때문에 인덱스가 0이에요. 그래서 인덱스 [0]이 맨 왼쪽 첫 칸을 가리키는 시작점이 됩니다.

탐색 알고리즘에는 여러 가지가 있는데, 가장 기본이 되는 두 가지를 배워볼 거예요.

📋 순차 탐색 (Sequential Search)

1️⃣ 개념

순차 탐색은 데이터의 맨 처음부터 차례대로 하나씩 비교하면서 찾는 방법이에요. 가장 단순하지만 확실한 방법이죠.

🎒 비유: 가방에서 색연필 찾기
가방 안 필통에서 빨간 색연필을 찾고 싶다고 해볼게요. 어떻게 찾나요?
"색연필을 하나씩 꺼내서 확인하기" → 빨강이면 멈추고, 아니면 다음 거 꺼내기.
이게 바로 순차 탐색이에요!

3️⃣ 동작 원리 정리

순차 탐색은 다음과 같이 동작해요.

  1. 배열의 인덱스 [0] 원소를 목표값과 비교한다.
  2. 같으면 찾았으니 멈춘다. 다르면 다음 원소로 이동한다.
  3. 2번을 반복한다. 끝까지 찾지 못하면 "없음"으로 결과를 낸다.

4️⃣ 비교 횟수

순차 탐색의 비교 횟수
최선: 1회  |  평균: n/2회  |  최악: n회

※ n은 데이터 개수예요. 운이 좋으면 1번에 찾고, 운이 나쁘면 끝까지 다 봐야 해요.

5️⃣ 장점과 단점

✅ 장점
  • • 알고리즘이 매우 단순해요
  • • 데이터가 정렬되어 있지 않아도 OK
  • • 어떤 자료에도 사용 가능
❌ 단점
  • • 데이터가 많으면 느려요
  • • 최악의 경우 n번 비교 필요

✂️ 이진 탐색 (Binary Search) 바로가기(영상)

1️⃣ 개념

이진 탐색은 가운데부터 비교하고, 목표값이 더 크면 오른쪽 절반만, 더 작으면 왼쪽 절반만 다시 검색하는 방법이에요. 매번 탐색 범위가 절반씩 줄어들어요.

⚠️ 정말 중요한 조건!
이진 탐색은 반드시 정렬된 데이터에서만 사용할 수 있어요. 정렬되지 않은 데이터에는 사용할 수 없어요!
📖 비유: 사전에서 단어 찾기
"행복"이라는 단어를 사전에서 찾는다고 해볼게요. 처음부터 한 장씩 넘기나요?
아니죠. 사전을 중간쯤 펼쳐서, "행복"이 그 페이지보다 앞쪽일지 뒤쪽일지 판단해요. 그리고 그 절반에서 또 가운데를 펼쳐서... 이게 바로 이진 탐색이에요!

2️⃣ 중간값 찾는 공식

이진 탐색의 핵심은 "가운데 위치를 어떻게 정하느냐"예요.

중간값 위치 (low와 high를 더한 뒤 2로 나눈 몫)
mid = (low + high) // 2
💡 "몫"이란?
나누었을 때 정수 부분만 사용한다는 뜻이에요(버림).
예) (1+8) ÷ 2 = 4.5 → 몫 4
예) (1+5) ÷ 2 = 3 → 몫 3

4️⃣ 동작 원리 정리

  1. 현재 탐색 범위의 왼쪽 끝(L)과 오른쪽 끝(R)을 정한다.
  2. 중간 위치 = (L+R) // 2 를 계산하고, 그 인덱스의 값과 목표값을 비교한다.
  3. 같으면 찾았으니 끝!
  4. 중간값이 목표값보다 작으면 → 왼쪽 끝을 중간 + 1로 바꿈 (오른쪽 절반만 검색)
  5. 중간값이 목표값보다 크면 → 오른쪽 끝을 중간 - 1로 바꿈 (왼쪽 절반만 검색)
  6. L > R이 되면 → 목표값이 배열에 없음

5️⃣ 비교 횟수

이진 탐색의 비교 횟수 (최악의 경우)
log₂n 번

※ 데이터를 매번 절반씩 줄이니까 비교 횟수도 적어요. 데이터가 8개면 최대 3번, 16개면 4번, 1024개여도 10번이면 충분!

6️⃣ 장점과 단점

✅ 장점
  • • 데이터가 많아도 매우 빠름
  • • 비교 횟수가 적음 (log₂n)
❌ 단점
  • • 반드시 정렬되어 있어야 함
  • • 알고리즘이 순차 탐색보다 복잡
